How It Works

  1. Read challenge.
  2. Write code to read from stdin, solve challenge, and print to stdout.
  3. ???
  4. PROFIT!

Rules

  • Don't try to break the system.
  • One person per username.
  • Submit original code only (no reusing old code for entire solution).
  • Don't just print the stdin, solve it manually, and write code that prints the solution.

Boring Details

  • Problems are ranked separately according to Development Speed, Speed Efficiency and Memory Efficiency.
  • Development Speed is ranked by order of first correct submission for each challenge.
  • Speed Efficiency is ranked by total execution time across all test cases (measured in seconds).
  • Memory Efficiency is ranked by total memory allocated during all all test cases (measured using HackerRank's black box).
  • You can submit an unlimited number of times, but only correct solutions will be counted for points.
  • Submitting with "Test" will give you helpful feedback with compiling/runtime errors and how much memory you're using.
  • Submitting with "Submit" will run our official input (never shown to you).
  • For each category, 100 points are awarded to the first place winner, 200 points to the second, etc. (minimum 50 points)
  • Points in the Efficiency categories may change as people continue to submit solutions.
  • Each challenge has a different multipliers for points earned in each category. Harder challenges will have higher multipliers.
  • Final prizes are given out ranked on overall score.
  • We use an API called Hacker Rank, which we reveresed engineered and are pretty sure uses a standard Amazon AWS EC2 running Ubuntu. Details about the code environment here.
